# 练习与输出

## 练习 1：解释三条扩展线

### 问题

MCP、插件、Skills 分别在 Claude Code 里解决什么问题？

### 参考答案

- MCP：协议型外部能力接入
- 插件：本地内容和命令扩展
- Skills：面向模型的结构化能力入口

## 练习 2：解释统一注册中心

### 问题

为什么多来源扩展必须回到统一注册中心？

### 参考答案

因为只有这样，系统才能统一处理过滤、排序、可见性和模型可调用性，避免能力集合碎片化。

## 练习 3：分析设计代价

### 问题

这套平台化扩展设计最大的复杂度代价是什么？

### 参考答案

扩展来源一多，调试链路、元数据治理和可见性判断都会明显复杂化。

## 练习 4：举一反三

### 问题

如果你自己的 AI 产品开始接第三方系统、支持用户自定义工作流，还想让模型主动调用这些能力，最先需要补什么？

### 参考答案

优先补统一抽象和统一注册中心，而不是继续堆 ad-hoc 集成。
