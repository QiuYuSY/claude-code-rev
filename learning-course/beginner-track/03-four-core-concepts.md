# 你只需要先懂的 4 个核心概念

很多小白一上来会被太多词吓住。

其实前期你只要先懂 4 个词，就够了。

## 1. Message

最简单理解：

系统里的“统一记录”。

它不只是聊天内容，还包括：

- 你的输入
- 模型的输出
- 工具执行结果
- 系统提示

## 2. Tool

最简单理解：

模型可以调用的能力。

比如：

- 读文件
- 改文件
- 跑命令

## 3. Query Loop

最简单理解：

系统的主循环。

它决定：

- 什么时候问模型
- 什么时候调工具
- 什么时候继续下一轮

## 4. AppState

最简单理解：

整个会话当前的状态总表。

里面会记住：

- 当前任务
- 当前设置
- 当前权限
- 当前插件/MCP 状态

## 先别急着懂的词

你现在可以先不深究：

- MCP
- compact
- bridge
- workflows
- attribution
- elicitation

先把前面 4 个词搞懂，后面就会容易很多。

## 记忆口诀

可以先记住这句：

`Message` 负责记  
`Tool` 负责做  
`Query Loop` 负责调度  
`AppState` 负责存当前状态
