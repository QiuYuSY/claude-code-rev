# 一次请求，简单讲明白

很多小白第一次看这种项目会卡住，因为一开始就掉进文件和函数名里。

所以这份文档只讲一件事：

“当你在 Claude Code 里输入一句话，到底发生了什么？”

## 极简版流程

```text
你输入一句话
  -> 系统把它变成消息
  -> 系统决定这轮该带哪些上下文
  -> 模型开始回答
  -> 如果需要，就调用工具
  -> 工具结果再回到消息里
  -> 系统继续下一轮，直到完成
  -> REPL 把过程和结果显示给你
```

## 把它理解成 5 个角色

### 1. 输入处理层

负责接住你的输入，不让它直接冲进模型。

### 2. 消息层

负责把“输入、输出、工具结果、系统提示”统一存起来。

### 3. query loop

这是系统的大脑，负责决定：

- 现在该不该问模型
- 现在该不该调工具
- 现在该不该继续下一轮

### 4. 工具层

负责真正执行：

- 读文件
- 改文件
- 跑命令
- 搜索

### 5. REPL 界面

负责把过程显示给你，不只是最后一句答案。

## 小白最该先理解的一点

Claude Code 不是“输入一句，输出一句”。

它更像：

- 一个会话
- 一个任务流
- 一个会自己调用工具的运行过程

## 看完这篇后，你应该会说

“原来 Claude Code 的核心不是聊天，而是一个会持续运转的任务推进循环。”
