# 案例概览

## 任务

用户在 Claude Code 中输入：

`帮我看看这个仓库的命令系统是怎么组织的`

## 系统会怎么处理

用最通俗的话说：

1. REPL 接住输入
2. 把输入变成统一消息
3. query loop 决定下一步要不要调工具
4. 系统可能去读文件、搜命令定义
5. 工具结果再回到消息里
6. 模型基于这些结果继续回答
7. REPL 把整个过程显示出来

## 这个案例为什么适合教学

因为它能同时覆盖：

- 启动后的交互入口
- 消息建模
- query loop
- 工具调用
- 文件分析
- 命令系统理解

但不会像“自动改文件”那样一开始就把风险和复杂度拉太高。
