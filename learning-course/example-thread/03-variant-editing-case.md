# 如果换成“修改一个函数”的案例，会多出什么复杂度

把案例换成：

`帮我修改一个函数，并验证它是否正常`

系统会比“分析仓库命令系统”多出几层复杂度。

## 新增复杂度 1：权限

因为这不再只是读文件，而是：

- 改文件
- 跑命令
- 可能影响代码库

所以权限和审批会变得更重要。

## 新增复杂度 2：状态变化

因为系统不再只是读结果，而是：

- 会产生新的文件状态
- 可能需要记录 file history

## 新增复杂度 3：验证链路

系统可能还要：

- 跑测试
- 看报错
- 再继续下一轮

这会让 query loop 更像真正的“任务推进器”。

## 教学意义

为什么这里不把“改函数”当默认案例？

因为它太早把：

- 权限
- 写操作
- 验证
- 多轮恢复

都压上来了。

对小白来说，“先分析仓库结构”更适合作为第一个固定案例。
