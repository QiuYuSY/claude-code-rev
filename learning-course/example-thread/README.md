# 固定案例线

这条线是为了解决一个问题：

“课程现在概念很多，但我总想知道它们到底在一个真实任务里怎么串起来。”

所以这里固定使用一个案例，贯穿整个课程。

## 固定案例

案例任务：

`让 Claude Code 帮我分析一个仓库的命令系统是怎么组织的`

为什么选这个案例：

- 它够真实
- 它能触发 REPL、消息、query loop、文件读取、命令系统分析这些主线
- 它不会一开始就把人拖进特别危险的写操作

## 阅读顺序

1. [案例概览](./01-case-overview.md)
2. [这个案例在课程各模块里分别发生了什么](./02-case-mapped-to-course.md)
3. [如果换成“修改一个函数”的案例，会多出什么复杂度](./03-variant-editing-case.md)

## 这条线的作用

它不是独立课程，而是“课程配套主线”。

你在看正式课程时，可以随时回到这里问自己：

- 我现在学的这个模块，在这个固定案例里是怎么发生的？
