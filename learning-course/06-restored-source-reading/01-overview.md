# 第六课概览

## 核心问题

这节课围绕 4 个问题展开：

1. 恢复版仓库和普通源码仓库，在阅读策略上有什么本质区别？
2. 为什么恢复版代码里最容易误读的是“兼容逻辑”？
3. 怎样把前 5 课学到的主干结构固化成自己的理解？
4. 怎样对这样一个复杂项目做有价值的工程评价？

## 你会读到哪些内容

- 仓库根目录的 `README.md`
- `shims/` 目录
- 前 5 课的配套图解
- 前 5 课中已经精读过的核心文件

## 先给结论

恢复版仓库最常见的误区不是“看不懂”，而是“看错重点”。

你真正应该优先学习的是：

- 启动主干
- query loop
- 工具抽象
- AppState
- MCP / plugin / skills 平台主线

而不是优先沉迷于：

- shim 细节
- fallback 细节
- 某个私有/原生能力的降级实现

## 这一课最值得学的地方

### 1. 学“如何读不完美源码”

真实工程里你经常拿到的并不是教科书式整洁仓库，而是有历史包袱、有恢复痕迹、有兼容层的复杂树。

### 2. 学“如何提炼架构主干”

真正的工程能力不只是读懂实现，而是能从噪音中提炼骨架。

### 3. 学“如何形成自己的设计判断”

你最终不应该只会说“这个项目很复杂”，而应该能说：

- 它最强的 5 个设计是什么
- 它最危险的 5 个复杂度来源是什么

## 推荐你带着什么视角读

读这节课时，反复问自己：

- 这段代码是在表达原始系统能力，还是在表达恢复策略？
- 这部分值得深入吗，还是只是过渡性兼容层？
- 我现在看到的是产品架构，还是恢复工程痕迹？

## 学完后的判断标准

如果你能完整说出这句话，方向就是对的：

“恢复版仓库依然很有学习价值，但前提是先抓架构主干，再理解兼容噪音，最后形成自己的设计评估，而不是被 shim 细节牵着走。”
