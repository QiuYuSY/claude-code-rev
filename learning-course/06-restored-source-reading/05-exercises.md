# 练习与输出

## 练习 1：解释恢复版价值

### 问题

为什么恢复版仓库依然值得学习？

### 参考答案

因为主干架构、抽象边界和运行时机制依然能被观察和提炼，兼容层并不会抹掉这些价值。

## 练习 2：解释主干与噪音

### 问题

你会如何向别人解释“架构主干”和“兼容噪音”的区别？

### 参考答案

主干是在回答系统本来如何运作，兼容噪音是在回答恢复后如何尽量把它补到可运行。

## 练习 3：做设计评价

### 问题

请列出 Claude Code 最值得借鉴的 5 个设计和最值得警惕的 5 个复杂度来源。

### 参考答案方向

借鉴：

- 启动分层
- query loop
- 工具运行时协议
- AppState
- 统一扩展注册中心

警惕：

- `main.tsx`
- `query.ts`
- feature gates
- 多来源扩展排查成本
- 恢复版兼容噪音
