# 第四课概览

## 核心问题

这节课围绕 4 个问题展开：

1. 为什么权限系统必须进入工具暴露和运行时决策层？
2. 为什么 Claude Code 要维护这么复杂的 message 结构？
3. 为什么上下文构建要单独建模？
4. 为什么权限、消息、上下文必须一起看？

## 你会读到哪些文件

- `src/utils/permissions/permissionSetup.ts`
- `src/main.tsx` 中权限初始化片段
- `src/utils/messages.ts`
- `src/context.ts`
- `src/Tool.ts`

## 先给结论

权限、消息、上下文不是三个独立专题，而是同一套运行时边界的三个面：

- 权限决定什么能做
- 消息决定发生过什么
- 上下文决定下一轮如何理解已经发生的事

## 这一课最值得学的地方

### 1. 学权限前置

Claude Code 并没有把权限理解成执行时的最后一道门，而是把它前置到了工具暴露和运行时模式切换里。

### 2. 学消息统一建模

这里的 message 不是聊天文本，而是承载工具结果、系统事件、恢复边界和摘要的一种统一结构。

### 3. 学上下文装配

上下文不是临时拼 prompt，而是显式地从 git、CLAUDE.md、日期、系统信息等来源装配出来。

## 推荐你带着什么视角读

读这节课时，反复问自己：

- 为什么权限初始化必须发生在 `getTools()` 之前？
- 为什么 `createUserMessage(...)` 这种工厂函数这么重要？
- 为什么 `getSystemContext()` 和 `getUserContext()` 要单独建模？

## 学完后的判断标准

如果你能完整说出这句话，方向就是对的：

“Claude Code 用权限控制能力边界，用消息统一运行时历史，用上下文装配决定模型输入，这三者共同保证 agent 运行时可控。”
