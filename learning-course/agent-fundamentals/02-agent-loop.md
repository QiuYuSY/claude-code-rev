# Agent 的最小闭环：感知、决策、执行、反馈

如果只学一个 Agent 设计概念，最应该学这个。

## 最小闭环

一个最简单的 Agent 闭环可以写成：

```text
感知 -> 决策 -> 执行 -> 反馈 -> 下一轮
```

## 1. 感知

Agent 先拿到输入：

- 用户目标
- 当前上下文
- 历史状态
- 外部信息

## 2. 决策

Agent 判断：

- 直接回答
- 先问问题
- 调工具
- 做计划
- 进入下一步

## 3. 执行

执行可能包括：

- 调模型
- 跑命令
- 读文件
- 写文件
- 调外部服务

## 4. 反馈

把执行结果重新带回系统里：

- 给用户看
- 给模型看
- 写入状态

## 为什么这个闭环重要

因为这就是 Agent 和普通单轮推理的本质区别。

很多系统看起来复杂，其实核心都是在扩展这个闭环。
