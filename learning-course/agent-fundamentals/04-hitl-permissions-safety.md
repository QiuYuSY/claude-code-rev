# Human-in-the-loop、权限与安全为什么是核心设计

难度：入门到中级
预计阅读时间：8-10 分钟
建议方式：必须读

很多初学者会误以为：

安全和权限只是最后补一个弹窗。

其实在 Agent 系统里，这通常是核心设计。

## 1. Human-in-the-loop 是什么

最简单理解：

- 人在关键节点保留干预权

比如：

- 允许/拒绝执行
- 修改计划
- 审查结果

## 2. 为什么 Agent 特别需要它

因为 Agent 不只是“回答”，它会：

- 动文件
- 跑命令
- 触发外部系统

一旦这些动作出错，成本远高于聊天答错一句话。

## 3. 权限为什么不是附属功能

权限会决定：

- 模型能不能看到某个能力
- 某个工具能不能自动执行
- 哪些动作必须确认

这就是为什么成熟 Agent 系统会把权限做到运行时里。

## 4. 对小白最重要的一句提醒

Agent 系统里，“安全”不是收尾工作，往往是架构的一部分。
