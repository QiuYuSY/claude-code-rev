# 第一课概览

## 核心问题

这节课围绕 4 个问题展开：

1. 进程从哪里开始？
2. 为什么入口要分层？
3. 什么东西应该在 `cli.tsx` 处理，什么东西应该在 `main.tsx` 处理？
4. 为什么命令系统必须有注册中心？

## 你会读到哪些文件

- `package.json`
- `src/bootstrap-entry.ts`
- `src/entrypoints/cli.tsx`
- `src/main.tsx`
- `src/commands.ts`

## 先给结论

Claude Code 的启动链路本质上是一个“性能感知的多层启动系统”。

它不是：

- 一个总入口文件
- 一个总 `main()`
- 一个简单命令数组

它是：

- 壳入口
- 分流入口
- 初始化编排器
- 多来源命令注册中心

## 学这节课最值钱的地方

### 1. 学启动分层

很多项目只有“入口”这个概念，没有“入口分层”这个概念。Claude Code 明显不是这样。

### 2. 学 fast-path 设计

真正成熟的 CLI，不会让 `--version` 这种命令去承担完整初始化成本。

### 3. 学注册中心思维

当命令来源变多后，系统必须从“手写列表”升级成“统一装配与过滤”。

## 推荐你带着什么视角读

读这节课时，不要只问“代码做了什么”，还要反复问：

- 为什么不放在别处做？
- 为什么现在做而不是更晚做？
- 为什么这一步是同步/异步？
- 为什么这一步要快？

## 本课学习成果

如果你学完后能用自己的话说出下面这句话，说明方向是对的：

“Claude Code 的入口不是一个文件，而是一条分层启动链路；它的命令系统不是命令列表，而是统一注册中心。”
