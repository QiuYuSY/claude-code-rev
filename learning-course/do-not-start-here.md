# 先别从这里开始

这份清单是专门写给新人的。

目的不是阻止你学习，而是避免你第一天就被劝退。

## 1. 不要先啃 `main.tsx`

原因：

- 它太大
- 分支太多
- 它假设你已经知道启动链路、权限上下文、REPL 和状态系统

更好的顺序：

- 先看第一课
- 再看启动链路图

## 2. 不要先啃 `query.ts`

原因：

- 它是系统心脏
- 但也是最复杂的状态机之一
- 如果你还没建立 Tool / Message / Query Loop 的心智模型，会直接迷路

更好的顺序：

- 先看 `agent-fundamentals`
- 再看第二课的概览和流程分析

## 3. 不要先钻 `shims/`

原因：

- 那里更多是恢复版兼容工程
- 不是 Claude Code 主干设计的最佳代表

更好的顺序：

- 先看第六课如何区分“架构主干”和“兼容噪音”

## 4. 不要把每个函数都当重点

原因：

- 真实系统不是每个函数都值得同样关注
- 课程真正想让你抓的是主线：
  - 启动
  - query loop
  - 状态
  - 权限
  - 扩展

## 5. 不要试图一周内全懂

原因：

- 这是成熟工程，不是 demo
- 第一遍的目标是建立骨架，不是掌握所有细节

## 6. 不要只背术语

原因：

- 背 `ToolUseContext`、`AppState` 这些词本身没用
- 你真正要理解的是它们解决了什么问题

## 最推荐的替代顺序

如果你现在很迷茫，就按这个顺序：

1. [从这里开始](./START-HERE.md)
2. [Agent 设计基础课](./agent-fundamentals/README.md)
3. [小白友好版入门路径](./beginner-track/README.md)
4. [固定案例线](./example-thread/README.md)
5. 再进入正式课程
