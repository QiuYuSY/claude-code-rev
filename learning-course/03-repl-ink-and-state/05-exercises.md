# 练习与输出

## 练习 1：解释 REPL 的角色

### 问题

为什么说 Claude Code 的 REPL 更像“驾驶舱”而不是“输入框”？

### 参考答案

因为它不仅显示消息，还承载任务、权限、插件、MCP、远程状态和多 agent 交互。

## 练习 2：解释 `AppState`

### 问题

为什么说 `AppState` 可以当作项目认知地图来读？

### 参考答案

因为它只收纳真正需要长期存在、跨模块共享、会影响交互和运行时的核心实体。

## 练习 3：分析设计代价

### 问题

这套 REPL + 状态中心设计最大的维护风险是什么？

### 参考答案

状态中心和交互编排层都容易持续膨胀，需要强边界管理，否则会变成新的理解瓶颈。

## 练习 4：举一反三

### 问题

如果你自己的终端 AI 产品开始支持多任务、远程状态和权限交互，最先要补的架构是什么？

### 参考答案

优先补统一状态中心和交互 orchestration 层，因为这些能力会迅速超出局部组件状态可管理的范围。
