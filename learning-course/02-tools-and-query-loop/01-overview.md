# 第二课概览

## 核心问题

这节课围绕 4 个问题展开：

1. 什么叫“工具抽象”？
2. 为什么 `ToolUseContext` 比一般项目里的 context 重得多？
3. 为什么 `query.ts` 不是“调用模型的工具函数”，而是运行时主循环？
4. 为什么工具、消息、预算、恢复逻辑会集中在同一个文件里？

## 你会读到哪些文件

- `src/Tool.ts`
- `src/tools.ts`
- `src/query.ts`
- `src/query/config.ts`
- `src/query/tokenBudget.ts`

## 先给结论

Claude Code 的核心不是“模型 + 若干工具”，而是：

- 一套工具协议
- 一套工具注册与过滤机制
- 一条以消息为主线的 query loop

换句话说，它不是“模型驱动工具”，而是“运行时驱动模型和工具共同协作”。

## 这一课最值得学的地方

### 1. 学会区分“工具接口”和“运行时插件接口”

很多项目里的工具只是函数调用，而 Claude Code 里的工具明显已经升级成运行时扩展点。

### 2. 学会看状态机式主循环

`query.ts` 最重要的价值，不是写法炫，而是它把复杂分支收束成一条统一主线。

### 3. 学会看消息闭环

用户消息、模型回复、工具结果、系统元消息最终回到同一条消息主线，这就是 agent runtime 的骨架。

## 推荐你带着什么视角读

读这节课时，反复问自己：

- 为什么工具结果必须回到消息系统？
- 为什么 `ToolUseContext` 要拿到这么多能力？
- 为什么不能把工具执行、预算控制、恢复逻辑拆散到很多外围模块？

## 学完后的判断标准

如果你能完整说出这句话，方向就是对的：

“Claude Code 不是在一次请求里顺便调工具，而是在 query loop 里把消息、工具、预算、恢复和继续决策统一编排。”
