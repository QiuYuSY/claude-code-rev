# 练习与输出

## 练习 1：定义工具系统

### 问题

Claude Code 里的工具系统，为什么不是“给模型几个函数”？

### 参考答案

因为工具会影响消息历史、权限行为、UI 状态、AppState 和下一轮 query 继续逻辑，所以它更像运行时插件接口。

## 练习 2：解释两层工具池

### 问题

为什么 `getAllBaseTools()` 和 `getTools(...)` 必须分开？

### 参考答案

因为系统需要同时保留：

- 平台理论上的全量能力
- 当前会话实际允许暴露的能力

## 练习 3：解释 query loop

### 问题

为什么 `query.ts` 更像状态机，而不是普通函数？

### 参考答案

因为它维护跨轮次状态、存在多条 continue 路径，并将“为什么继续”显式记录在 `transition` 中。

## 练习 4：分析设计代价

### 问题

这套设计最大的维护风险是什么？

### 参考答案

`query.ts` 和 `ToolUseContext` 都容易成为高复杂度中心，需要强测试和强边界管理。

## 练习 5：举一反三

### 问题

如果你自己的 AI 系统同时出现“流式输出、工具调用、中断恢复、多轮会话”，最应该优先补的架构是什么？

### 参考答案

优先补显式 query loop 和结构化消息系统，因为它们决定了主线是否可控。
