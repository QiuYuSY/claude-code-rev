# Unified Message Model Pattern

## 解决什么问题

当系统里既有聊天、又有工具、又有系统事件时，不能只靠文本历史。

## 模式概述

用统一消息模型承载：

- 用户输入
- 模型输出
- 工具结果
- 系统事件

## Claude Code 如何实现

核心落点：

- `src/utils/messages.ts`

## 优势

- 支持审计、恢复、回放

## 代价

- 消息结构会越来越复杂

## 适用场景

- 长会话
- 工具调用
- 多入口系统

## 不适用场景

- 只做一次性聊天

## 反例

如果只有字符串聊天记录，工具结果和系统状态就很难统一治理。
