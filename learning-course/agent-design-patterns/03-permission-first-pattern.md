# Permission-First Pattern

## 解决什么问题

模型不应该先看到所有能力，再在执行时临时被拦。

## 模式概述

先构建权限上下文，再暴露当前允许的能力集合。

## Claude Code 如何实现

核心落点：

- `src/utils/permissions/permissionSetup.ts`
- `src/tools.ts`

## 优势

- 推理路径更可控

## 代价

- 权限系统复杂度会上升

## 适用场景

- 有危险操作
- 有自动模式
- 需要审计

## 不适用场景

- 完全只读、完全低风险场景

## 反例

如果权限只在执行时审批，模型会一直尝试本不该看到的能力。
