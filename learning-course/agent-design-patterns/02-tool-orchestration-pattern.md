# Tool Orchestration Pattern

## 解决什么问题

Agent 有很多工具时，不能只靠“模型输出函数调用”粗暴处理。

## 模式概述

工具编排要统一处理：

- 工具池暴露
- 执行
- 结果回填
- 与消息和状态联动

## Claude Code 如何实现

核心落点：

- `src/Tool.ts`
- `src/tools.ts`
- `src/query.ts`

## 优势

- 工具行为更可控

## 代价

- 工具上下文会变重

## 适用场景

- 工具很多
- 工具影响会话状态

## 不适用场景

- 只有 1 到 2 个简单外部函数

## 反例

如果每个工具各写各的调用逻辑，系统很快会失去一致性。
